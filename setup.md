## What You Need

To participate in this workshop you will need:

1. **A laptop** with a web browser and internet access
2. **The challenge data files** (see below)
3. **A text editor** capable of opening plain text files (e.g. VS Code,
   Notepad++, TextEdit, or any editor you are comfortable with)

No specialist software is required. All exercises use files you can open in a
text editor and web browser.

## Download the Challenge Data

1. Download [challenge-data.zip][challenge-data-zip] and save it somewhere
   convenient (e.g. your Desktop or Documents folder).
2. Unzip/extract the file. You should end up with a folder called
   **`challenge-data`** containing numbered sub-folders for each exercise.

**Let your instructor know if you need help with this step.**

The challenge data folder has the following structure:

```bash
challenge-data/
├── 03_data-access-statements/
├── 04_fair-self-assessment/
├── 05_fix-this-folder/
├── 06_sharing-licensing-repositories/
├── 07_reproducibility-detective/
├── 09_metadata/
├── 10_find-a-repository/
├── 11_multi-technique-dmp/
└── 13_post-workshop-action-plan/
```

Each folder contains the files and instructions needed for that challenge.
Your instructor will tell you when to open each one.

[challenge-data-zip]: data/challenge-data.zip
